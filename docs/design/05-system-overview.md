# 系统设计概览

## 时间系统

### 120 时辰槽位

游戏内时间为 30 天，每天 4 个时辰：

```
天1:  [卯] [午] [酉] [子]
天2:  [卯] [午] [酉] [子]
...
天30: [卯] [午] [酉] [子]
```

每个时辰玩家选择**一个行动**（共 120 个选择点）。行动类型：

| 行动类型 | 说明 | 典型消耗 |
|----------|------|----------|
| 拜访 NPC | 前往某地与 NPC 互动 | 1 个时辰 |
| 修炼武学 | 提升已有武学熟练度 | 1 个时辰 |
| 探索事件 | 触发当前地区的随机/固定事件 | 1 个时辰 |
| 休息恢复 | 恢复气力和生命值 | 1 个时辰 |
| 战斗 | 回合制战斗（含剧情战和遭遇战） | 1-2 个时辰 |
| 远行 | 跨区域移动（需消耗多个时辰） | 2-3 个时辰 |

### 时辰特性

不同时辰可触发不同事件：

| 时辰 | 特有事件 | NPC 活跃度 |
|------|----------|------------|
| 卯时（晨） | 晨练偶遇、门派早课 | 铁骑营 NPC 活跃 |
| 午时（昼） | 正式拜访、任务触发 | 太虚/药王 NPC 活跃 |
| 酉时（暮） | 酒馆情报、社交活动 | 听雨阁/散人 NPC 活跃 |
| 子时（夜） | 潜行密会、危险遭遇 | 焚天教/影司 NPC 活跃 |

## 好感度系统

### 数值模型

每个 NPC 有四维好感度，每维 0-100：

```
变化量 = 基础变化 × 情境修正 × 关系修正
```

**基础变化：** 每个事件脚本预设的好感度变化值（如：+10 信任、-5 亲密）

**情境修正：** 当前时辰/地点/前置事件对变化量的调整
- 在 NPC 擅长的领域互动：×1.2
- 在 NPC 不擅长的领域互动：×0.8
- 有其他 NPC 在场：根据 NPC 间关系加减修正

**关系修正：** NPC 之间已有关系对好感度变化的影响
- 沈听雨在场时，苏青衡的信任变化 ×0.7（旧情缘尴尬）
- 厉无双在场时，清虚真人的任何变化 ×1.5（宿敌效应）

### 阈值效果

| 好感度范围 | 效果 |
|-----------|------|
| 0-20 | 敌对：NPC 主动回避，可能主动攻击 |
| 21-40 | 冷淡：仅公事公办，信息获取困难 |
| 41-60 | 正常：正常交互，可完成一般任务 |
| 61-80 | 友好：NPC 主动分享信息，赠送物品 |
| 81-100 | 深交：解锁专属剧情，NPC 愿意为玩家牺牲 |

### 一念效应

任何好感度变化都不是孤立的。核心规则：

1. **信任↑ 往往伴随 敬畏↓**（信任意味着亲近，亲近意味着少了距离感）
2. **亲密↑ 可能导致 忌惮↓**（亲密意味着了解，了解减少恐惧）
3. **敬畏↑ 往往导致 亲密↓**（敬畏意味着距离，距离侵蚀亲密）
4. **忌惮↑ 可能导致 信任↓**（恐惧滋生不信任）

这不是硬性规则，而是事件设计中的倾向。部分事件可以打破这些规律——这正是"一念"的精妙之处。

## 战斗系统

### 回合制框架

```
战斗流程：
┌─────────────────────────────┐
│ 1. 速度排序（基于角色属性） │
│ 2. 轮到该角色时选择行动：   │
│    - 使用武学（消耗气力）   │
│    - 防御（减少 50% 受伤）   │
│    - 使用道具               │
│    - 逃跑（部分战斗不可逃） │
│ 3. 计算伤害/效果            │
│ 4. 检查胜负条件             │
│ 5. 回到步骤 1               │
└─────────────────────────────┘
```

### 伤害公式

```
最终伤害 = 基础伤害 × 属性克制 × 气力加成 × 随机浮动(0.9-1.1)
```

- **基础伤害：** 武学招式的基础值
- **属性克制：** 克制 ×1.3 / 被克制 ×0.7 / 中性 ×1.0
- **气力加成：** 当前气力 / 最大气力 × 1.2（满气力时加成最大）

### 生命值与气力

| 属性 | 初始值 | 成长方式 |
|------|--------|----------|
| 生命值 | 100 | 每提升一级 +10（通过修炼获得经验） |
| 气力 | 100 | 装备/好感度加成 |
| 速度 | 10 | 固定值，受装备影响 |
| 攻击 | 10 | 每学会一招新武学 +2 |
| 防御 | 10 | 每提升一级 +1 |

### 战斗类型

| 类型 | 触发 | 特点 |
|------|------|------|
| 剧情战 | 关键剧情节点 | 固定敌人和数值，重叙事 |
| 遭遇战 | 随机事件 | 随机敌人，可逃跑 |
| NPC 挑战 | 敬畏/忌惮过高时触发 | NPC 主动挑衅玩家 |
| 终极战 | 第 30 天最终事件 | 高难度，决定结局 |

## 关系图系统

### 可视化需求

江湖关系图是一个可交互的力导向图（force-directed graph）：

- **节点：** 玩家 + 12 NPC，共 13 个节点
- **连线：** NPC 之间的关系（师徒/仇敌/旧友/情缘等）
- **连线颜色：** 正面关系 = 金色 / 负面关系 = 暗红 / 中性 = 灰色
- **连线粗细：** 关系强度
- **节点大小：** 与玩家当前好感度总和成正比
- **节点颜色：** 对应门派色或散人灰色

### 动态变化

- 关系图随游戏进度实时更新
- 新 NPC 发现时节点淡入
- 关系断裂时连线碎裂动画
- 玩家可点击任意节点查看该 NPC 详细好感度

## 存档与二周目

### 存档模型

不使用传统 save/load。采用"结局收集"模式：

- 每次游戏结束（触发任一结局）自动记录
- 二周目解锁"天眼模式"——可以查看每个抉择对结局的影响
- 已触发的结局在主菜单可回顾
- 30 天内可随时开始新档（覆盖旧档）

### 二周目差异

- 天眼模式：抉择选项旁显示结局影响图标
- 隐藏信息解锁：部分需要二周目才能触发的对话
- 真结局（一念成佛）仅在二周目可达
